# Security

This is a personal hobby project shared as-is under the MIT License (see [CONTRIBUTING.md](CONTRIBUTING.md)). There's no dedicated security contact and no formal disclosure process — Issues and PRs aren't actively monitored either.

If you find a problem, the most useful thing you can do is fix it in your own fork, since that's how this project is meant to be used. If you'd still like to flag something, open an issue or PR anyway; there's just no guarantee of a timely response.

## Supported versions

Only the latest code on `main` is supported. There are no maintained older versions or backports.

## Security model

This runs entirely on your own machine and LAN — there's no hosted service or third party involved beyond your own printer. Relevant design points, enforced in `bridge/bridge-server.mjs`:

- The bridge binds to `127.0.0.1` only; it's never reachable from the network.
- The bridge is stateless — your printer IP, API key, and username are sent to it fresh in each request from the widget's iCUE settings and are never written to disk or persisted between requests.
- The response cache is keyed by a hash of the printer IP and username, not the API key, so the key itself never ends up in the cache key or logs.
- `bridge/run-hidden.vbs`, generated by `install-autostart.ps1`, contains machine-specific paths and is git-ignored — don't commit it.

If you're forking this to add features, try to preserve those properties, especially "credentials are never written to disk by the bridge."
